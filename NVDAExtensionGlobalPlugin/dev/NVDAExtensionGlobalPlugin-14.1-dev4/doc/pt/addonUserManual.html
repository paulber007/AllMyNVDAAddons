1.2. Menu de configuração do extra

No menu "Preferências" do NVDA, é adicionado o submenu "Complemento Extensão de comandos do NVDA" que propõe, conforme o caso, as seguintes opções:

  • Sistematicamente o submenu "opções globais" que abre uma caixa de diálogo semelhante ao menu" Opções " do NVDA e que apresenta as seguintes categorias:
    • "Instalação das funcionalidades": permite ver o estado de instalação de cada funcionalidade fornecida pelo extra e, possivelmente, modificá-lo.
    • "Suplementos ao NVDA" para configurar as opções relacionadas ao NVDA fornecidas pelo extra,
    • "Computador" para configurar as opções relacionadas com a funcionalidade relativa ao computador (sistema operativo Windows, controlo de som) ,
      • "Teclado": para configurar as opções relacionadas ao teclado (remanência das teclas, Uso do teclado numérico),
    • "Avançado": opções para utilizadores experientes,
    • "Atualização": opções que controlam a atualização do extra.

  • Sistematicamente o submenu "opções de perfil de configuração atual" que abre uma caixa de diálogo semelhante ao menu" Opções " do NVDA e apresenta as seguintes categorias:
    • "Suplementos ao NVDA" para configurar as opções relacionadas ao NVDA fornecidas pelo extra, -" Analisador de texto": para definir as opções específicas para o analisador de texto.
  • "Renomear teclas do teclado "(se a funcionalidade estiver instalada): permite modificar, conforme necessário, os nomes das teclas do teclado que o NVDA fornece por padrão,
  • "Anúncio seletivo de teclas de comando "(se a funcionalidade estiver instalada): anuncia seletivamente o nome das teclas do teclado conforme necessário,
  • "Configurações de reinicialização" para redefinir as configurações para seu estado inicial durante a primeira instalação do extra.

Além do último menu, cada um está associado a um comando de script para ativá-lo. Usando a caixa de diálogo "definir comandos" do NVDA, é possível atribuir um a cada um deles.

Estes são os seguintes scripts com o seu comando do intérprete:

  • "Mostrar a caixa de diálogo de configuração do extra": "f1",
  • "Mostrar a caixa de diálogo de opções para o perfil de configuração atual": "control+f1",
  • "Mostrar o diálogo Renomear teclas do teclado":"f2",
  • "Mostrar o diálogo anúncio seletivo de teclas de comando": "f3".

Esta configuração é guardada de duas maneiras, dependendo da funcionalidade ou das opções em questão:

  • No ficheiro " NVDAExtensionGlobalPluginAddon.ini " colocado na pasta do utilizador "userConfig" do NVDA para a configuração usada, seja qual for o perfil de configuração,
  • No ficheiro NVDA.ini ou o ficheiro de perfil de configuração do perfil disparado no momento da alteração para a configuração a ser usada somente quando esse perfil for disparado.

O NVDA oferece a opção "guardar Configurações ao sair" (consulte o guia do utilizador do NVDA), que permite guardar automaticamente as configurações atuais do NVDA ao sair ou reiniciar o NVDA. Se esta opção não estiver marcada, as configurações serão salvas apenas por ação do utilizador ou pela modificação de um parâmetro que requer a reinicialização do NVDA.

Este extra segue este modo de operação.

Algumas alterações nas opções do extra exigem que as configurações sejam guardadas e o NVDA seja reiniciado. Nesse caso, é proposto ao utilizador para salvaguardar e reiniciar imediatamente como o NVDA faz quando há uma mudança de idioma.

Quando o extra é desinstalado, o seu ficheiro de configuração é excluído e as configurações do extra são removidas de todos os perfis de configuração.

Quando uma funcionalidade é desinstalada, as suas configurações, se as houver, são completamente removidas.

No caso de uma atualização, a instalação do extra oferece a possibilidade de manter (na medida do possível) ou não a configuração antiga.

Usando o submenu "configuração de reinicialização", é possível redefinir as configurações para o mesmo estado que quando o extra foi instalado pela primeira vez, incluindo o registado nos perfis de configuração.

Isso iniciará do zero sem ter que desinstalar e reinstalar o extra em caso de qualquer problema

1.4. Atualização

A atualização do extra padrão é feita automaticamente pouco tempo depois de iniciar o NVDA (entre 20 e 600 segundos) e no máximo uma vez por dia.

Pode desativar essa atualização desmarcando a opção " Verificar atualizações automaticamente "na categoria"atualização". A versão do extra pode ser:

  • Seja uma versão estável: seu número de versão é do tipo " 8.2 " por exemplo.
  • Seja uma versão de desenvolvimento: seu número de versão é então do tipo "8.2-dev1", ou "8.1-rc1", ou "8.1-beta2".

Por padrão, uma versão estável é atualizada com a versão estável mais recente (número da versão superior). No entanto, é possível atualizar a versão estável atual com uma versão de desenvolvimento da próxima versão estável. Basta marcar a opção "também atualizar versões estável para versões de desenvolvimento" localizado na categoria "atualização".

Uma versão de desenvolvimento é atualizada:

  • com uma versão de desenvolvimento mais recente,
  • com sua versão estável,
  • ou a versão estável mais recente se a opção anterior não estiver marcada.

O botão "Verificar atualização" na categoria "atualização" permite que procure imediatamente uma atualização.

O botão "Ver histórico" na mesma categoria permite visualizar o histórico da versão atual no navegador.

No caso de uma atualização automática, e se uma nova versão for detectada, a caixa de diálogo "Atualização" aparece para propor:

  • se a instalação imediata da nova versão,
  • seja para adiar a atualização para mais tarde. Nesse caso, a pesquisa será reiniciada pelo menos uma hora depois.

Para uma atualização de uma versão estável, o botão "o que há de novo" oferece a possibilidade, de abrir no navegador, o histórico do extra e assim poder conhecer as novidades da versão proposta.

1.5. Operação em Modo de segurança

Neste modo, o funcionamento do extra é degradado.

Como NVDA, não é possível salvar as alterações de configuração.

Algumas funcionalidades não estão disponíveis, tais como:

  • "Mostrar a lista de ícones na barra do sistema e janelas de aplicativos em execução",
  • "Registos do NVDA",
  • "Ferramentas de desenvolvimento".
  • Atualização do extra.

1.6. Compatibilidade

Como a configuração da funcionalidade "alterar perfil de voz" foi alterada desde a versão 9.0 do extra, ela será removida ao atualizar de uma versão anterior do extra. Todos os seletores são libertados e os perfis de voz associados são excluídos.

2.2. Instalação de uma funcionalidade

O extra permite instalar ou não instalar esta ou aquela funcionalidade. O estado de instalação de cada funcionalidade do extra é apresentado na categoria "instalação das funcionalidades" no diálogo "Complemento Extensão de comandos do NVDA-opções".

Este estado pode ser:

  • Não instalar: a funcionalidade não está instalada.
  • Instalar: a funcionalidade está instalada. Seus scripts podem ser usados usando os comandos do interpretador (veja a seção abaixo) e os comandos fornecido por padrão.
  • Instalar sem comandos: a funcionalidade está instalada. Os seus scripts podem ser usados usando os comandos do interpretador (veja a secção abaixo), mas nenhum comando é atribuído.

Por padrão, o extra não instala todas as funcionalidades.

++ Ativação dos scripts ++ [hdr0-2]

Em geral, cada script pode ser acionado de duas maneiras:

  • Seja com um comando clássico,
  • Usando o interpretador de comandos do extra (veja a secção abaixo).

Na caixa de diálogo "definir comandos" do NVDA, os scripts fornecidos pelo extra são colocados na sua própria categoria. A grande maioria dos scripts são divididos nas categorias: "Complemento Extensão de comandos do NVDA"," alterar perfil de voz"," Configurações"," Ferramentas "e"Voz".

Para scripts que podem ser iniciados por um comando do interpretador, isso é indicado após a descrição do script.

Dependendo da opção de instalação de uma funcionalidade ("Instalar, "ou" Instalar sem comando"), os comandos fornecidos por padrão são atribuídos ou não aos scripts dessa funcionalidade.

Por padrão, a grande maioria dos scripts que podem ser executados usando o interpretador de comandos não são atribuídos a nenhum comando.

++ Mostrar a lista de ícones na barra do sistema ++ [hdr1-1]

Este script originalmente criado por Yannick Maillot e Daniel Poiraud foi instalado usando o extra autônomo " ListeIconesZoneNotification_v1.2.nvda-addon " portanto, devemos desinstalá-lo.

A Sua descrição é: "Mostrar a lista de ícones na barra do sistema"

Ele é chamado, usando o comando "F11" do interpretador de comandos. É a contrapartida do script Jaws "Insert + f11".

Nenhum comando lhe é atribuído por padrão.

Mostra uma caixa de diálogo que contém:

  • A lista de ícones presentes na barra do sistema,
  • O botão "simples clique esquerdo" para ativar um simples clique esquerdo do rato,
  • O botão "simples clique direito" para ativar um simples clique direito do rato,
  • O botão "duplo clique esquerdo" para ativar um duplo clique esquerdo do rato,
  • O botão "Cancelar" para fechar a caixa de diálogo e terminar o script.

A navegação na caixa de diálogo é realizada por "Tab"ou" maiúscula + Tab " e na lista com as setas de direção para cima e para baixo.

Um toque na tecla "Escape" permite que feche a caixa de diálogo novamente.

Na lista de ícones, pressionando a tecla "Enter" ativa a pressão sobre o botão "simples clique esquerdo".

++ Mostrar a lista de janelas dos aplicativos em execução ++ [hdr1-2]

A descrição deste script é: "mostrar a lista de janelas dos aplicativos em execução com a possibilidade de alternar para um deles ou destruí-lo":

É a contrapartida do script Jaws "Insert + f10".

Ele é chamado usando o comando "KEY_WINDOUWS_LIST" do interpretador de comandos.

Nenhum comando lhe é atribuído por padrão.

Mostra uma caixa de diálogo que contém:

  • A lista de todos os aplicativos em execução,
  • O botão "mudar para" para colocar em primeiro plano a janela selecionada,

O objetivo destas ferramentas situa-se nesta condição e é:

  • Por um lado, para abordar o modelo "Adon scons template" para torná-lo fácil o suficiente para empacotar a extensão que terminamos de desenvolver,

2.40. Gestão de Configurações do utilizador

2.41. Apresentação

Esta funcionalidade visa permitir que o utilizador reinicie o NVDA com as configurações do utilizador de sua escolha.

Para isso, baseia-se na possibilidade de o NVDA oferecer ao utilizador a possibilidade de indicar o caminho do Diretório de configuração da linha de comando que o NVDA inicia.

Todos os parâmetros de configuração são guardados numa pasta específica no disco rígido e, por padrão, esta pasta é:

  • Para um NVDA instalado, na pasta "c:\Users\NomeUutilizador\AppData\Roaming" e é chamado de "nvda".
  • Para uma cópia portátil: na pasta programa desta cópia e é chamado de "userConfig".

Mas esta pasta de configuração pode ter outro nome e ir para outro lugar no disco rígido ou pen USB.

Para pedir ao NVDA para usá-la, basta indicá-la iniciando o NVDA com a linha de comando: nvda.exe-c "caminho da pasta de configuração"

Se o diretório não existir, o NVDA cria-o na inicialização e inicializa como-o quando é iniciado pela primeira vez após a instalação.

Caso contrário, o NVDA é iniciado usando os parâmetros desta pasta de configuração.

Portanto, é possível ter várias configurações de utilizador com o mesmo NVDA.

Por exemplo, isto permite que tenha, além da configuração básica, uma configuração de teste para testar novos extras ou uma configuração com um ajuste específico para uma situação específica.

Esta funcionalidade, portanto, fornece a possibilidade de:

  • Definir configurações do utilizador a serem usadas,
  • Inicializar novas configurações importando os parâmetros de outra configuração,

2.47. Análise de texto

Esta funcionalidade está em experimentação.

Ao modificar ou reler um documento usando apenas a fala, inconsistências, como parênteses sem correspondência, alterações de formatação indesejadas, espaços extras e sinais de pontuação parasitas ou não espaçados, geralmente podem estar ausentes.

A funcionalidade "análise de texto" instala uma ferramenta (Analisador) que avisa dessas irregularidades, usando uma mensagem de voz ou um ficheiro ONSAV ao reler por palavra ou por linha.

Pode configurar o analisador para anunciar essas irregularidades por número, descrição ou som.

A análise é dividida em três grupos de pesquisa:

  • Pesquisa de não-concordância entre os símbolos,
  • Pesquisa de anomalias: vários espaços contíguos, espaço ou tabulação no final da linha, pontuações isoladas ou mal espaçadas, letras maiúsculas ausentes ou mal colocadas, espaços ausentes ou inesperados" antes ou depois de uma pontuação,
  • Pesquisa de alterações de formato: Nome da fonte, cor, tamanho, atributos, estilo.

É possível desativar um grupo de pesquisa, bem como, individualmente, cada ação de um grupo.

Para isso, a categoria "Análise de texto" é adicionada na caixa de diálogo Opções do extra.

Os parâmetros nesta categoria são específicos para o perfil ativo que é anunciado quando a categoria é selecionada.

++ Ativar / desativar o analisador ++ [hdr31-1]

O analisador pode ser ativado ou desativado de três maneiras:

  • Através do comando" f6 " do intérprete,
  • Usando a caixa de seleção "Ativar analisador de texto" localizada na categoria "Análise de texto" da caixa de diálogo Opções do extra,
  • Usando o script "ativar ou desativar o analisador de texto" : não tem comando por padrão, mas é possível fazer isso usando a caixa de diálogo "definir comandos" do NVDA.

A ativação e desativação do analisador aplica-se apenas ao perfil de configuração ativo.

A caixa de seleção " Ativar o analisador de texto ao iniciar o extra "encontrada na categoria" análise de texto " fixa, para cada perfil, o estado de ativação do analisador de texto no lançamento do extra.

Com o analisador ativado, a análise é realizada a cada deslocamento do cursor de palavra em palavra, ou de linha em linha nos campos de edição de texto.

O resultado da análise é relatado de acordo com a escolha feita, por:

  • um sinal sonoro: a frequência e duração do sinal sonoro é configurável.
  • Uma mensagem de voz: por padrão, é "Alerta", mas pode ser alterado.
  • Um som escolhido de uma lista.
  • Uma mensagem vocal que dá a quantidade de irregularidades encontradas.
  • Uma mensagem vocal que dá a descrição detalhada das irregularidades encontradas.

Para configurações de alerta, consulte o grupo "Alerta" na categoria "Análise de texto".

++ Configurando a análise ++ [hdr31-2]

++ Pesquisa de não-concordância dos símbolos ++ [hdr31-2-1]

É ativado por padrão.

Alguns símbolos como parênteses, colchetes, teclas geralmente vão em pares: um símbolo de abertura, um símbolo de fechamento.

Esta pesquisa é para verificar esta correspondência.

Os possíveis símbolos são listados na lista " símbolos "do grupo" correspondência de símbolos "da categoria"Análise de texto".

Os símbolos na lista dependem do idioma escolhido para o NVDA. Eles são definidos no ficheiro " textAnalyzer.ini "colocado na pasta associada ao idioma da pasta "locale".

++ Pesquisa de anomalias ++ [hdr31-2-2]

Esta pesquisa é ativada por padrão.

A lista de anomalias A procurar é apresentada no grupo " anomalias "da categoria"Análise de texto".

Estas são:

  • Os múltiplos espaços contíguos,
  • Espaço ou tabulação no final,
  • Pontuações isoladas ou não espaçadas,
  • Letras maiúsculas ausentes ou invertidas,
  • O espaço obrigatório antes do símbolo,
  • O espaço obrigatório após o símbolo,
  • O espaço inesperado antes do símbolo,
  • O espaço inesperado após o símbolo.

Na lista, estas são caixas de seleção. Por padrão, todos elas estão marcadas. Ao desmarcar uma caixa de seleção, a verificação relativa não será feita.

Para anomalias referentes aos espaços em torno de um símbolo, Os símbolos envolvidos são relatados por marcação na lista que contém todos os símbolos previstos. Dependendo da necessidade, é possível marcar ou desmarcar este ou aquele símbolo. Os símbolos marcados por padrão são definidos para cada idioma do NVDA que o requer no ficheiro " textAnalysis.ini "colocado na pasta de idioma da pasta "locale".

Esta pesquisa está desativada por padrão.

As alterações de formato podem ser relatadas pelo analisador de acordo com o estado da caixa de seleção "relatar alterações" do grupo "Formato" da categoria "Análise de texto".

As mudanças de formato dizem respeito:

  • O tamanho da fonte,
  • Os atributos da fonte,
  • A cor,
  • O estilo.

Podem ser incluídos ou excluídos individualmente da pesquisa marcando-os ou desmarcando-os na lista" Tipos " do mesmo grupo.

2.49. Dispositivo temporário de saída de áudio

Este recurso está ativado por padrão.

Permite que altere temporariamente o dispositivo de saída de áudio atual para outro dispositivo de saída de áudio sem afetar as configurações do NVDA e isso para todos os perfis de configuração.

Cada perfil de configuração manterá os parâmetros configurados na categoria" Voz " usado pelo NVDA, incluindo o dispositivo de saída de áudio.

Vamos nomear este dispositivo, "Dispositivo temporário de áudio".

Quando um dispositivo temporário de áudio está ativo, o som de áudio (mensagens de voz e tons) emitido pelo NVDA será enviado para este dispositivo independentemente do dispositivo de saída de áudio do perfil de configuração ativo.

Os sons emitidos pelo sistema continuam a ser enviados para o dispositivo de saída de áudio selecionado nas configurações de Windows.

Como nada é alterado nas configurações do NVDA, o registo automático ou manual do mesmo não guarda nenhum vestígio deste dispositivo.

Para recuperar a operação normal, ou seja, o uso de dispositivos de saída de áudio dos perfis de configuração, será suficiente abortar o dispositivo de áudio temporário.

Para conseguir tudo isso, o extra instala vários scripts.

2.50. Orientação da saída de áudio do NVDA

Este recurso permite separar o som do NVDA do som dos aplicativos de áudio iniciados e, assim, ouvir num ouvido o som do NVDA e no outro ouvido o som de outros aplicativos de áudio.

Este recurso é instalado por padrão.

Tecnicamente, isso consiste em zerar o nível de volume dos canais de áudio direito ou esquerdo do NVDA e dos aplicativos.

O NVDA é uma aplicação monofônica, o mesmo som é reproduzido em ambos os canais. Não há perda de áudio fônico para colocar um canal em zero.

Mas esse não é o caso de aplicativos estereofônicos.

Para direcionar o som do NVDA, o extra instala os seguintes scripts:

  • Ouvir o NVDA à direita e o aplicativo sob o cursor à esquerda (comandos: "Flecha Derecha"),
  • Escutar o NVDA à direita e todos os aplicativos à esquerda (comandos: "control+Flecha Derecha"),
  • Ouvir o NVDA à esquerda e o aplicativo sob o cursor à direita (comandos: "Flecha Izquierda"),
  • Escutar o NVDA à esquerda e todos os aplicativos à direita (comandos: "control+Flecha Izquierda"),
  • Escutar o NVDA e a aplicação sob o cursor ao centro (comandos: "espacio"),
  • Escutar o NVDA e todos os aplicativos ao centro (comandos: "control+espacio")
  • Ouvir o aplicativo sob o cursor ao centro (comandos: "control+mayúscula+espacio")
  • Mostrar o gestor de áudio do NVDA e aplicativos ativos (comandos: "KEY_SOUND_SPLIT_MANAGE"R)

Nenhum comando é atribuído por padrão. Mas é sempre possível fazer isso usando o diálogo"definir comandos" do NVDA. Estes scripts são armazenados na categoria" Controle de som " desta caixa de diálogo.

O comando "control+mayúscula+espacio" é operacional mesmo que a funcionalidade não esteja instalada a fim, eventualmente, de poder reequilibrar o áudio de algumas aplicações que permaneceram desequilibradas após a desinstalação da funcionalidade.

A caixa de diálogo "gestor de áudio do NVDA e aplicativos ativos" também pode direcionar o som do NVDA, mas também definir o nível de volume dos dois canais de áudio do NVDA e aplicativos. apresenta:

  • A lista de aplicações de áudio,
  • O potenciômetro para alterar o volume do aplicativo selecionado,
  • A caixa de seleção para alterar o estado "som silenciado" do aplicativo selecionado,
  • O potenciômetro para alternar os canais esquerdo e direito do aplicativo selecionado.

Quando a lista está sob o cursor, os seguintes comandos são permitidos:

  • Ouvir o aplicativo selecionado à direita: "Seta para a direita",
  • Ouvir o aplicativo selecionado à esquerda: "seta para a esquerda",
  • Ouvi o aplicativo selecionado ao centro:"espaço",
  • Ouvir o NVDA à direita e o aplicativo selecionado à esquerda: "maiúscula + seta para a direita",
  • Ouvir o NVDA à esquerda e o aplicativo selecionado à direita: "maiúscula + seta para a esquerda",
  • Ouvir o NVDA e o aplicativo selecionado ao centro: "maiúscula + espaço",
  • Ouvir o NVDA à direita e todos os aplicativos à esquerda: "controle + seta para a direita",
  • Ouvira o NVDA à esquerda e todos os aplicativos à direita: "controle + seta para a esquerda",
  • Ouvir o NVDA e todos os aplicativos ao centro: "Controle + espaço",
  • Aumentar ou diminuir num passo, o volume do aplicativo selecionado: "control + seta para cima ou para baixo",
  • Aumentar ou diminuir num passo duplo, o volume do aplicativo selecionado: "control + page up ou Page Down",
  • Maximizar ou minimizar o volume do aplicativo: "controle + início ou fim",
  • Silenciar ou redefinir o som do aplicativo: "control + m",
  • Aumentar ou diminuir num passo, o volume principal: "control + maiúscula + seta para cima ou para baixo",
  • Aumentar ou diminuir num passo duplo, o volume principal: "control + maiúscula + page up ou Page Down",
  • Maximizar ou minimizar o volume principal: "control + maiúscula + início ou fim".

    Esta lista é atualizada automaticamente quando o cursor é dado.

    As alterações de alternância de canal ou de nível de volume são aplicadas imediatamente após cada comando.

    Por padrão, o nível de volume de aplicativos não pode ser maior que o nível de volume principal. Mas a opção" Aumentar o volume principal, se necessário " permite desativar essa limitação. Esta opção está no grupo " alteração de volume "na categoria" Computador "na caixa de diálogo" Opções " do extra.

    Uma proteção está prevista para evitar zerar simultaneamente o nível dos canais de áudio NVDA e que o seu nível de volume seja menor do que o seu limite de recuperação configurado.

    Quando um aplicativo é selecionado, o estado de equilíbrio dos canais é anunciado, bem como o estado ao silenciar o som.

Ao reiniciar o NVDA ou se o extra estiver desativado ou desinstalado, o NVDA será ouvido novamente em ambos os canais de áudio, enquanto os aplicativos permanecem em seu estado (armazenados pelo Windows).

Atenção: para uma operação correta, é necessário que o NVDA e os aplicativos de áudio usem o mesmo dispositivo de saída de áudio, como o Mapeador de som. Além disso, a funcionalidade tem apenas interesse que neste caso.

2.51. Ativação ou desativação rápida de extras

Para ativar ou desativar extras, o NVDA propõe nativamente o gestor de extras.

No entanto, em alguns casos, pode ser necessário desativar todo ou parte dos extras, deixando apenas alguns extras ativados.

Com o gestor de extras, é necessário revisar cada extra para ativá-lo ou desativá-lo.

Este extra tenta tornar essa manipulação mais rápida e, para isso, traz um script para mostrar a caixa de diálogo "ativação rápida de extras".

Este script é iniciado usando o comando" q " do interpretador. Nenhum comando está associado a ele. Mas é possível fazer isso usando a caixa de diálogo "definir comandos" do NVDA. Este está na categoria "Complemento Extensão de comandos do NVDA" com a descrição"Mostrar diálogo para ativar ou desativar rapidamente os extras".

Este diálogo apresenta:

  • A lista de extras instalados, ativados ou desativados,
  • O botão "Marcar tudo" para marcar todos os extras da lista,
  • O botão "Desmarcar tudo" para desmarcar todos os extras da lista,
  • O botão "guardar e reiniciar o NVDA" para validar as alterações e reiniciar o NVDA,
  • O botão "Cancelar" para fechar a caixa de diálogo sem qualquer modificação do estado de ativação dos extras.

A lista apresenta apenas extras ativados ou desativados ao mostrar a caixa de diálogo. Complementos incompatíveis, para serem ativados ou desativados na próxima inicialização, não fazem parte da lista.

Ao mostrar a caixa de diálogo, os extras ativados são marcados.

Para alterar o estado de ativação de um extra, basta marcá-lo para ativá-lo ou desmarcá-lo para desativá-lo.

As alterações só serão consideradas quando clicar no botão "guardar e reiniciar o NVDA".

4.3. Definir o atraso de espera de repetição para o mesmo comando

Alguns dos scripts do extra executam ações diferentes, dependendo do número de vezes que o comando associado ao script é composto. A repetição do comando deve ocorrer dentro de um curto período de tempo (500 ms por padrão). Para um script que executa três ações diferentes, o mecanismo é:

  • Na primeira pressão, o temporizador é iniciado,
  • Se não houver uma segunda pressão antes da expiração do temporizador, a primeira ação é executada. Caso contrário, o temporizador é reiniciado,
  • Se não houver uma terceira pressão antes da expiração do temporizador, a segunda ação é executada. Caso contrário, o temporizador pára e a terceira ação é executada.

Dependendo da habilidade do utilizador ou da configuração do hard onsare, essa duração pode ser muito longa ou muito curta. A opção intitulada " Duração da repetição do mesmo comando "na categoria" Avançado "da caixa de diálogo" Opções " do extra permite ajustar essa duração conforme necessário.
