  1. Введение + ++ Общие сведения ++ Это дополнение добавляет к глобальным командам NVDA следующие возможности:
  • Функциональности:
    • Отображение списка значков в области уведомлений или окон запущенных приложений,
    • Помощь в составлении сложных символов, таких как "e" в "o", перевёрнутый вопросительный знак и т.д.,
    • Озвучивание или отображение информации о приложении под курсором, например его версии, активного профиля конфигурации, загруженного дополнения,
    • Расширение возможностей виртуального буфера для браузеров Mozilla Firefox, Microsoft Internet Explorer, Microsoft Edge и Google Chrome:
      • новые команды навигации: абзац, раздел, якорь, основная область,
      • новые типы элементов, отображаемых в виде списка в диалоговом окне "Список элементов" (открывается с помощью NVDA+F7): (радиокнопка, абзац, фрейм, флажок и т.д.) и озвучивание количества элементов выбранного типа,
      • озвучивание адреса (URL) документа,
      • опция для циклической навигации при поиске следующего или предыдущего элемента,
      • скрипты для клика в текущей позиции объекта навигатора,
      • скрипты для таблиц:
        • озвучивание позиции текущей ячейки,
        • перемещение к следующей/предыдущей колонке/строке с чтением составляющих их ячеек,

    • Озвучивание предварительно выбранной папки в диалоговых окнах типа "Открыть", "Сохранить", "Сохранить как",

    • История голосовых объявлений,

        ++ Меню настройки дополнения ++[hdr-configurationMenu] В подменю "Настройки" NVDA добавлено подменю "NVDA global commands extension", которое предлагает, в зависимости от ситуации, следующие варианты:
  • всегда подменю "Глобальные параметры", открывающее диалог, похожий на диалог "Параметры" NVDA, с категориями:
    • "Установка функций": позволяет увидеть состояние установки каждой функции, предоставляемой дополнением, и при необходимости изменить его.
    • "Дополнения к NVDA": для настройки опций, связанных с NVDA, предоставляемых дополнением,
    • "Компьютер": для настройки опций, связанных с функциями, относящимися к компьютеру (операционная система Windows, управление звуком),
    • "Клавиатура": для настройки опций, связанных с клавиатурой (фиксация клавиш, использование цифрового блока),
    • "Дополнительно": опции для опытных пользователей,
    • "Обновление": параметры, управляющие обновлением дополнения.
  • всегда подменю "Параметры текущего профиля конфигурации", открывающее диалог, похожий на диалог "Параметры" NVDA, с категориями:
    • "Дополнения к NVDA": для настройки опций, связанных с NVDA, предоставляемых дополнением,
    • "Анализатор текста": для определения специфических опций анализатора текста.
  • "Переименование клавиш клавиатуры" (если функция установлена): позволяет изменить названия клавиш клавиатуры, заданные по умолчанию в NVDA, в соответствии с потребностями.
  • "Выборочное озвучивание клавиш клавиатуры" (если функция установлена): позволяет выборочно озвучивать названия клавиш клавиатуры по потребностям.
  • "Сброс конфигурации" для возврата конфигурации в исходное состояние при первой установке дополнения.

    Кроме последнего пункта, каждый связан со скриптом без команды активации. С помощью диалога "Команды" NVDA можно назначить им команду.

    Это следующие скрипты с их командами интерпретатора:
  • "Показать диалог настроек дополнения": "f1",
  • "Показать диалог настроек дополнения для текущего профиля конфигурации": "control+f1",
  • "Показать диалог переименования клавиш клавиатуры": "f2",
  • "Показать диалог для настройки выборочного озвучивания командных клавиш": "f3".

    Эта конфигурация сохраняется двумя способами в зависимости от функции или опции:
  • в файле "NVDAExtensionGlobalPluginAddon.ini", расположенном в папке пользователя "userConfig" NVDA, для конфигурации, используемой независимо от профиля конфигурации,
  • в файле NVDA.ini или файле профиля конфигурации профиля, активированного во время изменения, для конфигурации, которая будет использоваться только при активации этого профиля.

    NVDA предоставляет опцию "Сохранять конфигурацию при выходе" (см. руководство пользователя NVDA), которая позволяет автоматически сохранять конфигурацию NVDA при выходе или перезапуске. Если эта опция не отмечена, конфигурация сохраняется только по действию пользователя или при изменении параметра, требующего перезапуска NVDA.

    Это дополнение следует этому режиму работы.

    Некоторые изменения параметров дополнения требуют сохранения конфигурации и перезапуска NVDA. В этом случае пользователю предлагается немедленно сохранить и перезапустить, как это делает NVDA при изменении языка.

    При удалении дополнения его файл конфигурации удаляется, и конфигурация дополнения очищается из всех профилей конфигурации.

    При удалении функции её конфигурация, если она есть, полностью удаляется.

    В случае обновления установка дополнения предлагает возможность сохранить (по возможности) или не сохранять предыдущую конфигурацию.

    С помощью подменю "Сброс конфигурации" можно вернуть конфигурацию в то же состояние, что и при первой установке дополнения, включая конфигурацию, сохранённую в профилях конфигурации.

    Это позволяет начать с нуля без необходимости удаления и повторной установки дополнения в случае возможных проблем.

    ++ Контекстная помощь ++ В большинстве диалогов дополнения можно с помощью клавиши "F1" открыть руководство пользователя дополнения в наиболее подходящей главе.

    ++ Обновление ++[hdr-update] Обновление дополнения выполняется по умолчанию автоматически вскоре после запуска NVDA (от 20 до 600 секунд) и максимум один раз в день.

    Можно отключить это обновление, сняв отметку с опции "Автоматически проверять наличие обновлений" в категории "Обновление". Версия дополнения может быть:
  • либо стабильной: её номер версии имеет вид "8.2", например.
  • либо версией для разработчиков: её номер версии тогда имеет вид "8.2-dev1", "8.1-rc1" или "8.1-beta2".

    По умолчанию стабильная версия обновляется более новой стабильной версией (номер версии выше). Но можно обновить текущую стабильную версию до версии для разработчиков следующей стабильной версии. Для этого достаточно отметить опцию "Обновлять стабильные версии до версий для разработчиков" в категории "Обновление".

    Версия для разработчиков обновляется:
  • более новой версией для разработчиков,
  • своей стабильной версией,
  • или последней стабильной версией, если предыдущая опция не отмечена.

    Кнопка "Проверить наличие обновлений" в категории "Обновление" позволяет немедленно проверить наличие обновлений.

    Кнопка "Просмотреть историю" в той же категории позволяет просмотреть историю текущей версии в браузере.

    В случае автоматического обновления, если обнаружена новая версия, появляется диалоговое окно "Обновление", предлагающее:
  • либо немедленно установить новую версию,
  • либо отложить обновление. В этом случае проверка возобновится не менее чем через час.

    В случае обновления до стабильной версии кнопка "Что нового" позволяет открыть в браузере историю дополнения и узнать о нововведениях предлагаемой версии.

    ++ Работа в безопасном режиме ++ В этом режиме работа дополнения ограничена.

    Как и NVDA, невозможно сохранить изменения конфигурации.

    Некоторые функции больше недоступны, например:
  • "Отображение списка значков в области уведомлений и окон запущенных приложений",
  • "Журналы NVDA",
  • "Инструменты разработки",
  • обновление дополнения.

    ++ Совместимость ++ Конфигурация функции "Переключение голосового профиля" изменилась, поэтому она очищается при обновлении с версии дополнения старше 9.0. Все селекторы освобождаются, и связанные голосовые профили удаляются.

  1. Функциональности +[hdr0] ++ Введение ++[hdr0-1] +++ Установка функциональности +++ Это дополнение позволяет устанавливать или не устанавливать ту или иную функциональность. Состояние установки каждой функциональности дополнения представлено в категории "Установка функциональностей" диалога "Параметры".

    Это состояние может быть:
  • Не устанавливать: функциональность не установлена.
  • Установить: функциональность установлена. Её скрипты можно использовать с помощью команд интерпретатора (см. следующую главу) и командных жестов, предусмотренных по умолчанию.
  • Установить без командного жеста: функциональность установлена. Её скрипты можно использовать с помощью команд интерпретатора (см. следующую главу), но никакой командный жест им не назначен.

    По умолчанию это дополнение не устанавливает все функциональности.

    +++ Активация скриптов +++[hdr0-2] Обычно каждый скрипт можно активировать двумя способами:
  • либо с помощью стандартного командного жеста,
  • либо с использованием интерпретатора команд дополнения (см. следующую главу).

    В диалоговом окне "Команды" NVDA скрипты, предоставляемые дополнением, размещаются в собственной категории. Большинство скриптов распределены по категориям: "NVDA global commands extension", "Переключение голосового профиля", "Настройка", "Инструменты" и "Речь".

    Для скриптов, которые можно запустить командой интерпретатора, эта команда указана после описания скрипта.

    В зависимости от выбора установки функциональности ("Установить" или "Установить без командного жеста"), командные жесты, предусмотренные по умолчанию, назначаются или не назначаются скриптам этой функциональности.

    По умолчанию большинство скриптов, которые можно запустить с помощью интерпретатора команд, не связаны ни с каким командным жестом.

    +++ Интерпретатор команд +++[hdr0-1-3] Он запускается с помощью командного жеста "NVDA+j". При этом слышен звуковой сигнал.

    Затем можно активировать скрипты функциональностей с помощью команд.

    На сегодняшний день эти команды не изменяемы.

    Команда "h" интерпретатора перечисляет все скрипты установленных функциональностей, доступных через команду интерпретатора. В этом списке можно запустить выбранный скрипт, нажав клавишу "Enter".

    Команда "g" интерпретатора открывает руководство пользователя дополнения.

    Интерпретатор команд останавливается:
  • при нажатии клавиши "Escape",
  • либо автоматически, если введена команда. Если эта команда не соответствует ни одному скрипту, звуковой сигнал сообщает об этом.

    ++ Отображение списка значков в области уведомлений и списка окон запущенных приложений ++[hdr1] Эта функциональность по умолчанию установлена с командными жестами.

    Она добавляет следующие скрипты:
  • Отображение списка значков в области уведомлений,
  • Отображение списка окон запущенных приложений,

  • поиск предыдущего элемента продолжается с конца документа, если достигнуто начало.

    Переход через начало или конец документа сопровождается звуком и голосовым сообщением.

    Опция "Циклическая навигация" в категории "Дополнения к NVDA" диалога "Параметры" дополнения позволяет включить или отключить этот режим.

    Эта опция отображается только если установлена функциональность "Расширение возможностей виртуального буфера".

    По умолчанию эта опция не отмечена. Сохраняется базовое поведение NVDA.

    ++ Скрипты для клика в текущей позиции объекта навигатора ++[hdr5-6] Эти скрипты позволяют выполнить левый или правый клик мышью, одинарный или двойной, в текущей позиции объекта навигатора.

    Они фактически объединяют два типа скриптов NVDA:
  • скрипт "Переместить указатель мыши к текущему объекту навигатора": сочетание "NVDA + / на цифровой клавиатуре" (для настольных компьютеров) или "NVDA + Shift + m" (для ноутбуков),
  • и скрипты для клика мышью:
    • одинарный левый клик: "/ на цифровой клавиатуре" (для настольных компьютеров) или "NVDA + ù" (для ноутбуков с французским языком NVDA) или "NVDA+[" (для ноутбуков с другим языком),
    • одинарный правый клик: "* на цифровой клавиатуре" (для настольных компьютеров) или "NVDA+*" (для ноутбуков с французским языком NVDA) или "NVDA+]" (для ноутбуков с другим языком).

      Командные жесты этих скриптов:
  • "nvda+,": выполнить одинарный левый клик в позиции объекта навигатора. Двойное нажатие выполняет двойной клик.
  • "nvda+majuscule+,": выполнить одинарный правый клик в позиции объекта навигатора. Двойное нажатие выполняет двойной клик.

    Эти командные жесты можно изменить, и они находятся в категории "Мышь" диалога "Команды" NVDA.

    ++ Выборочное озвучивание командных клавиш ++[hdr13] Эта функциональность установлена по умолчанию.

    NVDA позволяет озвучивать командные клавиши с помощью опции "Озвучивать командные клавиши" в категории "Клавиатура" диалога "Параметры" NVDA или с помощью команды клавиатуры "NVDA+4".

    Эта функциональность добавляет, через настройку и с помощью диалогового окна, возможность:
  • не озвучивать некоторые командные клавиши, когда опция "Озвучивать командные клавиши" отмечена,
  • озвучивать некоторые командные клавиши, когда опция "Озвучивать командные клавиши" не отмечена.

    Для отображения этого диалога есть три возможности:
  • выбрав и активировав пункт "Выборочное озвучивание командных клавиш" в подменю "NVDA global commands extension",
  • с помощью команды интерпретатора "f3",
  • с помощью скрипта "Показать диалог для настройки выборочного озвучивания командных клавиш", предусмотренного без командного жеста. Командный жест можно назначить с помощью диалога "Команды" NVDA.

    Диалог содержит:
  • флажок для выбора режима, в котором настраиваются клавиши,
  • список возможных клавиш,
  • список возможных комбинаций клавиш-модификаторов с дополнительными клавишами "NVDA", "Alt", "Control", "Shift", "Windows".

    Нажатие пробела отмечает или снимает отметку с выбранной клавиши или комбинации.

    Если флажок "Озвучивать командные клавиши" в диалоговом окне не отмечен, отметка клавиши позволяет всё равно озвучивать эту клавишу, когда NVDA настроен не озвучивать их.

    Список комбинаций клавиш-модификаторов тогда доступен, и все элементы этого списка отмечены по умолчанию.

    Чтобы клавиша, нажатая с определённой комбинацией, не озвучивалась, достаточно снять отметку с этой комбинации.

    Если элемент "нет" в списке комбинаций отмечен, клавиша будет озвучена даже при нажатии без комбинации клавиш-модификаторов.

    С другой стороны, если флажок "Озвучивать командные клавиши" отмечен, отметка клавиши позволяет не озвучивать эту клавишу, когда NVDA настроен их озвучивать.

    Список комбинаций клавиш-модификаторов тогда доступен, и все элементы этого списка отмечены по умолчанию. Это означает, что клавиша, нажатая с комбинацией или без, не будет озвучена.

    Снятие отметки с комбинации позволяет тогда озвучивать клавишу с этой комбинацией.

    Она позволяет сделать залипающими клавиши NVDA ("Insert" на цифровой клавиатуре, обычной или расширенной, клавишу "Caps Lock") и клавиши-модификаторы ("Shift", "Control", "левый Alt", "правый Control Alt", "Windows").

    Цель — возможность использовать, например, только один палец для составления жеста команды, требующего нажатия нескольких клавиш.

    Для активации или деактивации залипания клавиш достаточно нажать одну из следующих комбинаций клавиш:
  • "Правый Shift", "Правый Control", "правый Shift",
  • "Левый Shift", "левый Control", "левый Shift".

    Залипание клавиш может автоматически активироваться при запуске NVDA, если отметить флажок "Активировать залипание клавиш при запуске NVDA" в группе "Режим залипания клавиш".

    Активация или деактивация залипания озвучивается.

    Когда залипание активно, нажатие на одну из клавиш NVDA или модификаторов запускает таймер, и клавиша запоминается. Этот таймер настраивается с помощью опции "Длительность залипания клавиш" в группе "Режим залипания клавиш".

    Также можно ограничить залипание только клавишей "NVDA", отметив флажок "Только клавиша NVDA залипает" в той же группе. По умолчанию этот флажок не отмечен, и все клавиши-модификаторы залипают.

    Начало и конец залипания сигнализируются звуковым сигналом, который можно отключить с помощью следующих флажков в категории "Режим залипания клавиш":
  • "Подавать звуковой сигнал при начале залипания клавиши",
  • "Подавать звуковой сигнал при окончании залипания клавиши".

    Они отмечены по умолчанию.

    Во время этого таймера любое новое нажатие на одну из этих клавиш перезапускает таймер, и новая клавиша также запоминается, но звуковой сигнал не подается.

    Нажатие любой другой клавиши на клавиатуре выполняет жест команды, составленный из комбинации всех запомненных клавиш с этой последней клавишей.

    Каждый раз, когда эта последняя клавиша будет нажата снова во время таймера, жест команды будет выполнен повторно.

    Если эта клавиша нажата вне таймера, она выполняется отдельно, и запомненные клавиши забываются.

    Внимание: Необходимо дождаться окончания залипания, чтобы обрабатывать каждую новую клавишу отдельно. Например: "Alt" для отображения меню, а затем переход по пунктам меню.

    Однако можно остановить залипание до окончания таймера, повторно нажав последнюю залипшую клавишу.

    Оно не активно по умолчанию. В группе "Режим залипания клавиш" предусмотрен флажок "Специальное залипание для gmail.com".

    На этой странице можно вводить сочетания клавиш для выполнения команд, но некоторые из них требуют последовательного нажатия двух букв. В частности, буквы "g", "h", "*" должны быть нажаты перед другой буквой. Эта последовательность должна быть достаточно быстрой для выполнения связанной команды. Это создает проблемы для людей, у которых недостаточно ловкости.

    Цель этого залипания — запомнить первую букву, пока не будет нажата вторая.

    Когда первая буква нажата, запускается таймер (слышен глухой звуковой сигнал), и буква запоминается.

    Нажатие второй буквы в течение настроенного времени залипания отправляет обе буквы в браузер, и таймер останавливается.

    Если новая буква не нажата, первая буква отправляется в браузер по истечении таймера (слышен высокий звуковой сигнал).

    Чтобы NVDA отправляла буквы в браузер, необходимо сначала переключиться в режим формы.

    ++ Управление пользовательскими жестами команд ++[hdr20] Эту функцию нельзя удалить.

    Диалог содержит:
  • классическую иерархию, отображающую жесты команд по категориям и каждый скрипт с его описанием,
  • кнопку "Удалить" (активна при выборе жеста) для отмены добавления или удаления выбранного жеста,
  • кнопку для отмены всех добавлений или удалений жестов команд.

  • категория называется: "Имя модуля Python». Имя класса Python" (например: appModules.notepad++.editWindow.EditWindow),
  • описание добавленных жестов команд называется: "Имя скрипта Python" (например: goToPreviousBookmark).

    ++ Эмуляция клавиши "Applications" ++[hdr25] Для компенсации отсутствия клавиши "Applications" на некоторых компьютерах и учитывая, что сочетание "Shift+F10" может не выполнять ту же функцию, дополнение устанавливает скрипт без жеста команды в категории "Эмулируемые клавиши системной клавиатуры" с описанием "Эмулировать нажатие клавиши: Applications".

    Для его использования достаточно назначить ему жест команды или клавишу клавиатуры, которая редко используется, например, клавишу "Print Screen", клавишу "Pause".

    ++ Закрытие всех окон ++[hdr26] Дополнение устанавливает скрипт для закрытия всех открытых окон.

    Этот скрипт находится в категории скриптов дополнения с описанием "Закрыть все открытые окна".

    Он запускается с помощью команды "k" интерпретатора. Для него не назначен жест команды.

    ++ Список работающих дополнений ++[hdr27] Менеджер дополнений или Add-on Store отображает список всех установленных дополнений и их состояние: включено или выключено.

    Но даже если дополнение включено, оно может не загружаться NVDA. В принципе, журнал NVDA сигнализирует об этом, но ни менеджер дополнений, ни Add-on Store этого не показывают.

    Может быть полезно в какой-то момент узнать, какие дополнения действительно работают.

    Для этого дополнение устанавливает скрипт с описанием "Отобразить список работающих дополнений" в категории дополнения в диалоге "Жесты команд".

    Этот скрипт запускается с помощью команды "e" интерпретатора. Для него не назначен жест команды.

    Дополнения перечислены по типам:
  • Глобальное дополнение: это дополнение, которое предоставляет дополнительные функции доступности независимо от приложения под курсором.
  • Дополнение для приложения: это дополнение, которое предоставляет дополнительные функции доступности для конкретного приложения.
  • Драйвер синтеза речи: это дополнение, которое устанавливает драйвер для синтеза речи.
  • Драйвер брайлевского дисплея: это дополнение, которое устанавливает драйвер для брайлевского дисплея.
  • Смешанное дополнение: это дополнение, которое сочетает в себе различные типы дополнений, перечисленные выше.

    ++ Инструменты разработчика ++[hdr22] Эта функция по умолчанию не установлена.

    Она устанавливает скрипт с описанием "Отобразить диалог с инструментами для разработки дополнений".

    Этот скрипт находится в категории "Инструменты" диалога "Жесты команд" NVDA.

    Он запускается с помощью команды "t" интерпретатора. Для него не назначен жест команды.

    Активация этого скрипта отображает диалог, содержащий:
  • список установленных дополнений,
  • кнопку для обновления версии выбранного дополнения в файле buildVars.py,
  • кнопку для обновления файла "manifest.ini" выбранного дополнения с информацией из его файла "buildVars.py",
  • кнопку для создания файла каталога (.pot) для выбранного дополнения,
  • кнопку для подготовки выбранного дополнения (выполняет несколько действий за один раз),
  • список языков, поддерживаемых выбранным дополнением,
  • кнопку для создания файла "manifest.ini" локализации для одного или всех поддерживаемых языков,
  • кнопку для создания HTML-документации для выбранного дополнения,
  • кнопку для создания файла "buildVars.py" на основе информации из файла manifest.ini,
  • кнопку для открытия папки выбранного дополнения в проводнике.

    +++ Основные понятия +++ Для упаковки дополнения (создания файла установки дополнения) рекомендуется использовать шаблон "nvda adon scons template". Этот шаблон использует файл "buildVars.py", который определяет для дополнения всю необходимую информацию, такую как: внутреннее имя, версия, имя и описание для перевода на каждый поддерживаемый язык. Этот шаблон также используется для перевода строк дополнения с помощью системы перевода, реализованной сообществом "NVDA-addon.org".

    Цель этого шаблона — после завершения разработки дополнения, с помощью простой команды и после заполнения файла "buildVars.py":
  • создать файл "manifest.ini",
  • создать все файлы "manifest.ini" локализации,
  • скомпилировать все файлы .po,
  • создать HTML-документацию,
  • наконец, создать файл установки дополнения .nvda-addon.

    При создании дополнения с нуля ничего этого нет. Необходимо самостоятельно создать файл "manifest.ini", возможно, файл "manifest.ini" локализации и файл документации.

    Цель этих инструментов — приблизиться к шаблону "adon scons template", чтобы упростить упаковку дополнения после завершения разработки, а также протестировать все или часть перед использованием шаблона для создания дополнения.

    +++ Создание файла buildVars.py +++[hdr22-1] Первый шаг — создать файл "buildVars.py" на основе информации из файла "manifest.ini", который был создан для разработки дополнения.

    Кнопка "Создать файл buildVars.py" запускает эту операцию.

    Только этот файл будет изменяться в дальнейшем для обновления файла "manifest.ini" (например, изменение версии или описания).

    Для использования с шаблоном "adon scons template" его нужно дополнить информацией, которой нет в файле "manifest.ini".

    +++ Обновление версии дополнения +++[hdr22-2] Кнопка "Обновить версию дополнения" позволяет изменить версию, указанную в файле buildVars.py выбранного дополнения.

    Она отображает диалоговое окно с полем ввода, инициализированным текущей версией из файла.

    Нажатие кнопки "OK" сохраняет новую версию в файле.

    +++ Обновление файла manifest.ini +++[hdr22-3] Файл "manifest.ini" должен быть обновлен при каждом изменении файла "buildVars.py".

    Используйте кнопку "Обновить файл manifest.ini" для этого.

    +++ Создание файла "manifest.ini" локализации +++[hdr22-4] Любое изменение информации "addon_summary" или "addon_description" в файле "buildVars.py" требует перевода этого описания на каждый поддерживаемый язык дополнения (учет в файле nvda.po и компиляция этого файла).

    Чтобы менеджер дополнений или Add-on Store отображал эту информацию для выбранного языка NVDA, необходимо создать или обновить файл "manifest.ini" локализации для этого языка.

    Кнопка "Создать файл manifest.ini локализации" позволяет это сделать. Можно обновить файл для выбранного языка или для всех языков.

    +++ Создание HTML-документации +++[hdr22-5] Файлы документации дополнения могут быть в формате "markdown" (.md) или "txt2tags" (.t2t).

    Для отображения пользователю в браузере эти файлы должны быть преобразованы в HTML.

    Это можно сделать просто с помощью кнопки "Создать HTML-файлы документации" после выбора дополнения в списке дополнений и выбора языка или всех языков в списке языков.

    При открытии в браузере эти HTML-файлы требуют файлов стилей (style.css для файлов Markdown и style_t2t.css для файлов txt2tags), размещенных в папке doc дополнения.

    +++ Создание файла каталога +++[hdr22-6] Кнопка "Создать файл POT" позволяет создать файл каталога для выбранного дополнения.

    После создания этот файл размещается в папке "locale\en" дополнения и называется "имя-версия.pot", где "имя" — имя дополнения, а "версия" — его версия.

    +++ Подготовка дополнения +++[hdr22-7] Кнопка "Подготовить дополнение" запускает для выбранного дополнения выполнение следующих действий:
  • обновление файла "manifest.ini" на основе файла "buildVars.py",
  • создание файла "manifest.ini" локализации для каждого поддерживаемого языка,
  • создание HTML-документации,
  • создание файла каталога.

    +++ Открытие папки дополнения +++[hdr22-8] Кнопка "Открыть папку дополнения" открывает проводник Windows в папке выбранного дополнения.

    ++ Управление пользовательскими конфигурациями ++[hdr30] +++ Обзор +++ Эта функция позволяет пользователю перезапустить NVDA с выбранной пользовательской конфигурацией.

    Для этого она использует возможность NVDA указывать путь к папке конфигурации в командной строке запуска NVDA.

    Все параметры конфигурации сохраняются в определенной папке на жестком диске, и по умолчанию эта папка находится:
  • для установленной NVDA: в папке "C:\Users\ИмяПользователя\AppData\Roaming" и называется "nvda".
  • для портативной версии: в папке программы этой версии и называется "userConfig".

    Но эта папка конфигурации может иметь другое имя и находиться в другом месте на жестком диске или USB-накопителе.

    Чтобы NVDA использовала ее, достаточно указать ее при запуске NVDA с командной строкой: nvda.exe -c "путь к папке конфигурации"

    Если папка не существует, NVDA создает ее при запуске и инициализирует, как при первом запуске после установки.

    В противном случае NVDA запускается с параметрами из этой папки конфигурации.

    Таким образом, можно иметь несколько пользовательских конфигураций с одной и той же NVDA.

    Это позволяет, например, иметь, помимо базовой конфигурации, тестовую конфигурацию для проверки новых дополнений или конфигурацию с определенными настройками для конкретной ситуации.

    Эта функция предоставляет средства для:
  • определения пользовательских конфигураций, которые будут использоваться,
  • инициализации новых конфигураций путем импорта параметров из другой конфигурации,
  • запуска NVDA с выбранной конфигурацией без необходимости использования командной строки.

    +++ Управление папками пользовательских конфигураций +++[hdr30-1] Прежде всего, необходимо определить конфигурации, которые будут использоваться.

    Для этого дополнение устанавливает скрипт "Отобразить диалог для управления пользовательскими конфигурациями", который запускается с помощью команды интерпретатора "n".

    Для него не назначен жест команды, но можно назначить его с помощью диалога "Жесты команд" NVDA.

    Эта команда отображает диалог "Менеджер пользовательских конфигураций", который содержит:
  • список папок конфигураций, которые можно использовать,
  • кнопку "Перезапустить NVDA с этой папкой конфигурации",
  • кнопку "Изменить или подготовить эту папку",
  • кнопку "Добавить новую папку",
  • кнопку "Добавить существующую папку",
  • кнопку "Удалить эту папку",
  • кнопку "Очистить эту папку",
  • кнопку "Открыть эту папку",
  • кнопку "Закрыть" для закрытия диалога.

    Список содержит все папки, добавленные пользователем, и папку, автоматически добавленную дополнением.

    Папка активной конфигурации озвучивается и отмечается символом "✓" перед ее именем.

    При первом отображении этого диалога дополнение автоматически добавляет в список текущую папку конфигурации и сохраняет ее.

    Для этой папки кнопки "Удалить эту папку" и "Очистить эту папку" отсутствуют.

    Новая папка, которая еще не является папкой конфигурации, отмечается знаком вопроса.

    Папка конфигурации NVDA характеризуется своим содержимым. В частности, она должна содержать:
  • папку "addons",
  • папку "profiles",
  • папку "updates",
  • файл "nvda.ini".

    Кнопка "Изменить эту папку" отсутствует, если выбрана папка активной конфигурации.

    ++++ Добавление папки ++++[hdr30-1-1] Кнопки "Добавить новую папку" и "Добавить существующую папку" позволяют добавить папку в список папок конфигураций.

    Нажатие первой кнопки отображает диалог "Добавление новой папки", который содержит:
  • поле ввода для указания идентификатора папки,
  • поле ввода для указания пути к папке,
  • кнопку "Обзор..." для выбора расположения папки,
  • кнопку "Добавить" для подтверждения и добавления папки в список,
  • кнопку "Отменить".

    Идентификатор папки позволяет назвать папку следующим образом: "nvdaUserConfig-идентификатор".

    Под этим именем папка отображается в списке и на жестком диске или USB-накопителе.

    Нажатие второй кнопки отображает диалог "Добавление существующей папки", который содержит:
  • поле ввода для указания полного пути к существующей папке,
  • кнопку "Обзор..." для выбора этой папки,
  • кнопку "Добавить" для подтверждения,
  • кнопку "Отменить".

    ++++ Удаление папки ++++[hdr30-1-2] Кнопка "Удалить папку" позволяет удалить выбранную папку из списка и с диска.

    Эта кнопка отсутствует, если выбрана папка активной конфигурации.

    Подтверждение удаления запрашивается для этой операции.

    ++++ Подготовка или изменение папки ++++[hdr30-1-3] В зависимости от того, является ли папка папкой конфигурации или нет, диалог отображает:
  • либо кнопку "Изменить папку",
  • либо кнопку "Подготовить эту папку".

    Их цель — предоставить инструменты для импорта параметров из другой папки конфигурации.

    Активация этих кнопок отображает соответствующий диалог, который отличается только заголовком.

    В этих диалогах представлены:
  • список папок, из которых будет выполняться импорт (только папки конфигурации),
  • кнопка "Импортировать дополнения",
  • кнопка "Импортировать словари речи",
  • кнопка "Импортировать произношения символов или пунктуации",
  • кнопка "Импортировать изменения жестов команд",
  • кнопка "Импортировать профили конфигурации",
  • кнопка "Импортировать параметры NVDA (обычная конфигурация)",
  • кнопка "Импортировать всю конфигурацию",
  • кнопка "Закрыть" для завершения диалога.

    Имя папки, которая будет изменена, указано в заголовке диалога.

    Действие каждой кнопки относится к содержимому папки, выбранной в списке.

    Для импорта дополнений можно импортировать все или часть дополнений из выбранной папки и сохранить или удалить дополнения, уже включенные в изменяемую папку. То же самое для импорта профилей.

    Импорт всей конфигурации позволяет полностью дублировать выбранную папку и получить идентичную копию конфигурации этой папки.

    ++++ Перезапуск NVDA с другой конфигурацией ++++[hdr30-1-4] Кнопка "Перезапустить NVDA с этой папкой" позволяет перезапустить NVDA с конфигурацией из выбранной папки.

    Эта папка может быть:
  • либо полной папкой конфигурации: NVDA перезапустится с параметрами из этой папки.
  • либо неполной папкой (отмечена знаком вопроса): NVDA перезапустится, инициализируя эту папку и используя параметры, которые были в нее импортированы.

    Чтобы эта другая конфигурация также могла использовать возможность перезапуска, необходимо, чтобы это дополнение было в ней установлено.

    В этом случае не нужно заново добавлять папки конфигурации. Дополнение восстановит ранее сохраненные папки.

    Это сохранение выполняется в файле "NVDAExtensionGlobalPlugin-userConfigs.ini", который хранится в:
  • папке программы NVDA, если это портативная версия,
  • папке "AppData\Roaming" пользователя, если это установленная NVDA.

    ++++ Открытие папки конфигурации ++++[hdr30-1-5] Кнопка "Открыть эту папку" позволяет открыть выбранную папку конфигурации в проводнике.

    +++ Примеры использования +++ ++++ Запуск NVDA с новой пустой конфигурацией ++++
  • Нажать "NVDA+j", затем "n" для отображения диалога "Менеджер пользовательских конфигураций".
  • Нажать кнопку "Добавить новую папку".
  • Ввести идентификатор папки конфигурации (например, "МояКонфигурация"),
  • Выбрать расположение папки конфигурации: по умолчанию это папка программы NVDA, но она может быть в другом месте на жестком диске или даже на USB-накопителе.
  • Нажать кнопку "Добавить": папка появляется в списке с знаком вопроса и выбирается.
  • Нажать "Перезапустить NVDA с этой папкой конфигурации".

    NVDA перезапустится и инициализирует конфигурацию, как при первом запуске после установки.

    ++++ Запуск NVDA с идентичной конфигурацией ++++ После первых 5 шагов:
  • Нажать кнопку "Подготовить эту папку" для отображения диалога.
  • Нажать кнопку "Импортировать всю конфигурацию" и подождать, так как это может занять некоторое время.
  • Нажать кнопку "Перезапустить NVDA с этой папкой конфигурации".

    ++++ Запуск NVDA с существующей конфигурацией ++++
  • Нажать "NVDA+j", затем "n".
  • Нажать кнопку "Добавить существующую папку".
  • Выбрать путь к папке конфигурации: она может находиться даже на USB-накопителе.
  • Нажать кнопку "Добавить": папка появляется в списке и выбирается.
  • Нажать "Перезапустить NVDA с этой папкой конфигурации".

    ++++ Перенос активной конфигурации с одного компьютера на другой через USB-накопитель ++++

    ++ Анализ текста ++[hdr31] Эта функция находится в стадии эксперимента.

    При редактировании или проверке документа с использованием только речи такие несоответствия, как несопоставленные скобки, непреднамеренные изменения форматирования, лишние пробелы и лишние или неправильно расположенные знаки пунктуации, часто могут быть пропущены.

    Функция "Анализ текста" устанавливает инструмент (анализатор), который предупреждает об этих несоответствиях с помощью голосового сообщения или WAV-файла при проверке по словам или строкам.

    Можно настроить анализатор так, чтобы он объявлял эти несоответствия по количеству, описанию или звуку.

    Анализ разделен на три группы поиска:
  • поиск несопоставленных символов,
  • поиск аномалий: множественные пробелы подряд, пробел или табуляция в конце строки, изолированные или неправильно расположенные знаки пунктуации, пропущенные или неправильно расположенные заглавные буквы, пропущенные или лишние пробелы до или после знаков пунктуации,
  • поиск изменений форматирования: название шрифта, цвет, размер, атрибуты, стиль.
  • поиск орфографических ошибок.

    Можно отключить группу поиска, а также индивидуально каждое действие в группе.

    Для этого в диалоге настройки дополнения добавлена категория "Анализ текста".

    Параметры этой категории специфичны для активного профиля, который объявляется при выборе категории.

    +++ Активация / деактивация анализатора +++[hdr31-1] Анализатор можно активировать или деактивировать тремя способами:
  • с помощью команды "f6" интерпретатора,
  • с помощью флажка "Активировать анализатор текста" в категории "Анализ текста" диалога настройки дополнения,
  • с помощью скрипта "Активировать или деактивировать анализатор текста": для него не назначен жест команды, но можно назначить его с помощью диалога "Жесты команд" NVDA.

    Активация и деактивация анализатора касается только активного профиля конфигурации.

    Флажок "Активировать анализатор текста при запуске дополнения" в категории "Анализ текста" устанавливает для каждого профиля состояние активации анализатора при запуске дополнения.

    Результат анализа сигнализируется в соответствии с выбранным вариантом:
  • звуковым сигналом: частота и длительность сигнала настраиваются.
  • голосовым сообщением: по умолчанию это "Предупреждение", но можно изменить.
  • звуком, выбранным из списка.
  • голосовым сообщением с количеством найденных несоответствий.
  • голосовым сообщением с подробным описанием найденных несоответствий.

    Для настройки предупреждения см. группу "Предупреждение" в категории "Анализ текста".

    ++ Направление вывода звука NVDA ++[hdr34] Эта функция позволяет разделить звук NVDA и звук других аудиоисточников, чтобы слышать звук NVDA в одном ухе, а звук других источников — в другом. Затронуты только приложения с двухканальным аудиовыходом, использующие одно и то же аудиоустройство.

    Также она позволяет регулировать громкость тональных сигналов NVDA (когда Wasapi не используется, возможно для версий NVDA до 2025.1) и изменять громкость звуковых файлов NVDA и дополнения.

    Функция установлена по умолчанию.

    Технически переключение звука вправо или влево заключается в установке уровня громкости правого или левого аудиоканалов NVDA и других источников звука на ноль. Для центрирования звука оба канала устанавливаются на одинаковый уровень. Этот уровень равен наибольшему из уровней двух каналов. Для переключения звука с одной стороны на другую уровень канала, установленный на ноль, принимается за уровень для другого канала, если тот также на нуле.

    NVDA — монофоническое приложение, поэтому один и тот же звук воспроизводится на обоих каналах. Нет потери звука при установке одного канала на ноль.

    Однако это не относится к стереофоническим приложениям.

    Для направления звука NVDA дополнение устанавливает следующие скрипты:
  • Слышать NVDA справа, а звук приложения под курсором — слева (команда: стрелка вправо),
  • Слышать NVDA справа, а звук всех приложений — слева (команда: control+стрелка вправо),
  • Слышать NVDA слева, а звук приложения под курсором — справа (команда: стрелка влево),
  • Слышать NVDA слева, а звук всех приложений — справа (команда: control+стрелка влево),
  • Слышать NVDA и звук приложения под курсором по центру (команда: space),
  • Слышать NVDA и звук всех приложений по центру (команда: control+space),
  • Слышать звук приложения под курсором по центру (команда: control+shift+espace),
  • Показать менеджер аудиоисточников (команда: f8).

    По умолчанию для них не назначены жесты ввода, но их можно задать через диалог "Жесты ввода" NVDA. Эти скрипты находятся в категории "Управление звуком".

    Команда "control+shift+espace" работает даже при отключенной функции, чтобы можно было восстановить баланс звука для некоторых приложений, оставшихся несбалансированными после удаления функции.

  • список звуков, используемых NVDA и дополнением,
  • кнопку "Воспроизвести" для прослушивания выбранного звука,
  • список выбора громкости: положительное значение усиливает звук, отрицательное — уменьшает,
  • кнопку "Тест" для прослушивания результата изменения громкости,
  • кнопку "Применить громкость" для применения громкости к выбранному звуку,
  • кнопку "Применить громкость ко всем звукам" для применения громкости ко всем звукам,
  • кнопку "Сбросить выбранный звук" для восстановления исходного состояния звука,
  • кнопку "Сбросить все звуки" для восстановления всех звуков.

    Если выбранный звук не изменялся, кнопка "Сбросить звук" не отображается.

    Если в списке нет измененных звуков, кнопка "Сбросить все звуки" не отображается.

    Звук запуска NVDA нельзя изменить, так как он воспроизводится до загрузки дополнения.

    ++ Обнаружение орфографических ошибок ++[hdr104] NVDA может обнаруживать орфографические ошибки:
  • при вводе текста, когда слово набирается,
  • при чтении: перемещение по строкам, словам или команда "Прочитать все".

    Для этого используются две опции NVDA:
  • опция "Орфографическая ошибка" в группе "Информация о документе" категории "Форматирование документов",
  • опция "Воспроизводить звук при орфографических ошибках во время ввода" в категории "Клавиатура".

    Чтобы NVDA обнаруживал ошибки при чтении и вводе, необходимо включить опцию "Орфографические ошибки". Ошибки будут объявляться как "Орфографическая ошибка".

    Чтобы ошибки при вводе обозначались звуком, необходимо включить опцию "Воспроизводить звук при орфографических ошибках во время ввода".

    Дополнение изменяет это поведение следующим образом:
  • обнаружение ошибок при чтении и вводе независимы. Если опция "Орфографические ошибки" отключена, но опция "Воспроизводить звук при орфографических ошибках во время ввода" включена, ошибки будут обозначаться при вводе,
  • для каждой ситуации (чтение или ввод) можно настроить способ обозначения ошибок: двойной сигнал, звук NVDA, сообщение "Орфографическая ошибка" или объявление неправильно написанного слова с использованием специального профиля.

    Выбор способа обозначения ошибок осуществляется в группе "Обнаружение орфографических ошибок" категории "Дополнения к NVDA" глобальных настроек дополнения с помощью выпадающих списков "При вводе обозначать как" и "При чтении обозначать как".

    Доступные варианты:
  • звуковой сигнал: воспроизводится двойной сигнал,
  • звук: воспроизводится звук NVDA,
  • сообщение: объявляется "Орфографическая ошибка",
  • объявление слова: слово объявляется с параметрами голоса из специального профиля "Объявление неправильно написанных слов", добавленного дополнением. Этот профиль автоматически активируется при объявлении ошибки.

    Профиль "Объявление неправильно написанных слов" связан с уникальным триггером и не отображается в списке триггеров. Его не следует связывать с другими триггерами.

    Для скрипта с тремя действиями механизм следующий:
  • при первом нажатии запускается таймер,
  • если второе нажатие не происходит до истечения таймера, выполняется первое действие, иначе таймер перезапускается,
  • если третье нажатие не происходит до истечения таймера, выполняется второе действие, иначе таймер останавливается и выполняется третье действие.

    Опция "Временной интервал для повторения жестов ввода" в категории "Дополнительно" диалога "Настройки" дополнения позволяет настроить этот интервал.

  • клавиши "Home" и "End",

    ++ Ограничение повторения клавиш ++[hdr307]

    Когда клавиша нажата и не отпущена достаточно быстро, Windows генерирует серию нажатий, что может мешать, особенно в поле ввода.

    Эта функция пытается ограничить количество повторных нажатий до минимума.

    Она управляется флажком "Ограничивать повторения" в группе "Повторение клавиш" категории "Дополнительно" глобальных настроек дополнения и интервалом повторения, который настраивается в той же группе.

    Когда опция включена, все повторные нажатия той же клавиши, происходящие быстрее заданного интервала, игнорируются.

    По умолчанию опция отключена, а интервал повторения установлен на 100 мс.

    Невозможно оставить только первое нажатие, так как интервал до следующего нажатия слишком велик (около 500 мс).

    Клавиши-модификаторы не затрагиваются этим ограничением.
