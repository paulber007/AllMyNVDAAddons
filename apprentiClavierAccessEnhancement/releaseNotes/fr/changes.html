1. 1.14 (26/02/2026)

Compatibilité avec NVDA 2026.1.

Les versions de NVDA antérieures à la 2025.1 ne sont plus supportées.

Correction de bogues :

  • la lecture des pages d'explication se fait maintenant correctement lorsque le synthétiseur OneCore est utilisé.

Développement :

  • suppression du code concernant les versions de NVDA inférieures à la 2025.1.
  • amélioration de la réactivité de l'annonce de la lettre ou du mot suivant.
  • pour afficher le "Quoi de neuf" d'une nouvelle version de l'extension, le document "changes.html" n'est plus ouvert par"urlopen", mais est téléchargé , enregistré momentanément dans le dossier "appData\local\temp" et ouvert dans le navigateur. - prise en compte de la fermeture du site rawgit.com pour la mise à jour des extensions.
  • utilisation de json en lieu et place de pickle pour stoquer l'état de la mise à jour de l'extension.

Mise en conformité avec l'API de NVDA :

  • prise en compte du déplacement de la version de NVDA dans le module "buildVersion.py".

2. 1.13 (05/06/2025)

Compatibilité avec NVDA 2025.1

Les versions de NVDA antérieures à la 2024.1 ne sont plus supportées.

Correction de bogues :

  • la recherche de mise à jour ne se fait plus lorsque l'option "Rechercher automatiquement les mises à jour " est décochée juste après le démarrage de NVDA.

Développement :

  • lors de l'installation de l'extension, la configuration courante n'est plus sauvegardée dans le dossier de l'extension mais est maintenant sauvegardée dans le dossier de la configuration utilisateur. Cela permet si une installation se passe mal de conserver quand même la configuration courante.

Mise en conformité avec l'API de NVDA :

  • utilisation de la méthode "ExecAndPump" du module "systemUtils ".
  • prise en compte de l'obsolescence de gui.messageBox et gui.message.messageBox, remplacés par MessageDialog:
    • MessageDialog est maintenant utilisé pour toutes les versions de NVDA supérieures à 2025.1.
